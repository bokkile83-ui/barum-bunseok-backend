## 0. 통신 규칙
결론→근거→실행. 단정체. 위로·감성 금지. 옵션 3개 이내. 서두·반복 금지.
제출 전: 셀프렌더→육안검수→셀프팩폭 먼저. 통과한 것만 제출. 미통과 시 "완료" 금지, 어디가 틀렸는지 자진보고. 모르면 빈칸+[확인], 추측 금지. 안내는 항상 클릭 가능한 직접 링크로.

## 1. 핵심 등식
- 등식1: 한장보장표 = 엑셀 담보값 = PPT 담보값. (별첨·요약표 이중대조)
- 등식2: PPT는 완성 엑셀만 읽는다. PPT 합계 = 엑셀 데이터셀(C~끝열-1) 직접 합산(=SUM 캐시 의존 금지).

## 2. 입력·동작
- 입력 = 사용자가 Adobe Acrobat으로 변환한 .txt 1개 → 엑셀 1개 + PPT 2개(보장분석지·치료비) 자동.
- PDF 직접 스캔·파싱 금지. 줄단위 직독. 1인 1파일, 새 고객 업로드 시 이전 값 전부 0.
- 고객명 = txt 파일명 우선, 없으면 내용("○○○ 고객님").
- 행 삭제·이동·재정렬·폰트변경·새 레이아웃 금지. 추가만. 마스터 열어 값만 채움.

## 3. 추출 엔진
- .txt 줄단위 직독. llm_extract(Claude API) + rule_extract(폴백). API 죽어도 rule로 돈다.
- 회사명 3단 복원: 목록 회사열→별첨 제목→상품명. 없으면 "미확인". 발행사(let:/롯데)를 타계약에 전파 금지.
- 소스 순서: ①별첨 보장현황(정본) ②세부가입현황(교차) ③한장보장표(검산).
- 자동화 천장 ≈ 90%. 깨진 별첨(금액열 분리 등) = [확인] → 신인 수기.

## 4. 제외 보험 (이 4가지만 제외, 그 외 전부 포함)
①실효계약 ②미납해지 ③농업인NH안전보험('농업인' 표기된 것만) ④자동차보험.
NH농협생명·손보 = 일반보험 포함. 치아보험·운전자상해보험 포함.

## 5. 보험 건수·합계
- 최대 50건. 계약 = 열(C부터). 합계는 항상 맨 끝 열.
- 각 담보 행 = 계약 가로 SUM(동적, 하드코딩 금지). 보험료 합계 = 가로 SUM, 숫자만.
- 수술비 1~5종 2행은 슬래시 합산(숫자 SUM 예외).

## 6. 갱신/비갱신 판정 (순서)
1) 담보명 "갱신형" 명시 → 갱신
2) 만기 9999(종신) → 비갱신
3) 총회차 240 초과 → 비갱신
4) 납입기간 == 보장기간 동일 → 갱신 / 다르면 비갱신
운전자도 동일. **일상배상책임은 무조건 갱신(ALL).**

## 7. 수술비 1~5종 (슬래시 정본)
- 상해 종수술비 1행 + 질병 종수술비 1행 = 총 2행. 종별 슬래시(예 20/50/100/500/1000).
- 다계약 종별 합산 슬래시. 끝열 합계도 종별 합산 슬래시. PPT 동일.

**8.3.2 PPT 심장 클러스터 주입 (2026.06.25, 실파일 검증)**
PPT 보장분석지 심(心) 박스의 협심증·심부전·염증·부정맥은 **떠있는 단일 텍스트박스 `TextBox 4`(클러스터 박스)** 하나에 묶여 들어간다. 라벨박스(`TextBox 53='심장'`)가 아니다.
- 주입 형식: `협심증/심부전/염증 [값] / 부정맥 [값]` (엑셀 37·38·39행=동일값→앞 묶음 1개 숫자, 40행=부정맥).
- **빈맥 제외**(미적용 규칙 일관). 값은 엑셀에서 동기화, **한글단위·콤마**(예 700→`700`만 단위 맥락, 1,000→`1천`) 적용.
- 급성심근(심장Ⅱ)=안쪽 원 박스, 허혈성=중간 원 박스, 산정특례(심장)/혈전용해/2대주요치료비=상단 리스트 박스. (이들은 정상 주입 확인됨)
- ★**버그(현 앱 v27):** 마스터 폼의 클러스터 박스에 **예시 잔존값**(`협심증/심부전/염증/빈맥1000 / 부정맥 500만`, 뇌혈관 `1000`)이 박혀 있는데, 앱이 이를 **클리어/주입하지 않아** 김선우 실값(700/300, 뇌혈관 500)과 불일치. → 주입 전 잔존값 클리어 필수.
- ★전(前)/후(後) 두 슬라이드 **모두 동기화**. (현재 전 슬라이드는 클러스터 박스 공란, 후 슬라이드는 잔존값 — 둘 다 오류)
- 등식 재확인: 엑셀 심장블록(37~45행) = PPT 심 박스. 엑셀이 정답, PPT가 따라와야 함.
