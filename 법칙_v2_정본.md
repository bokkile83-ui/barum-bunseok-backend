# 바름 보장분석 법칙 v2 최종 (2026.06.05 확정)

## ★ 최종 목표 — 두 등식
표(PDF) = 엑셀 입력 / 엑셀 입력 = PPT 입력
(등식1: 별첨·요약표 이중읽기 자동대조로 검증 / 등식2: PPT는 완성된 엑셀만 읽는 구조로 원천보장)

## Ⅰ. 기본 동작
1. PDF 1개 → 분석 → 엑셀 → PPT 자동. 1인 1파일, 새 고객 업로드 시 이전 고객 자동삭제, 최대 50건.
2. 고객명 = PDF 파일명 우선, 없으면 PDF 내용.
3. 표 행 삭제 금지, 추가만. 표본 골격 불변.

## Ⅱ. 분석 순서·대조
4. 보유계약 목록 → 계약별 별첨 1장씩 → 계약별 세로 컬럼. A담보가 B회사로 가면 안 됨 — 계약별 대조 필수.
5. 현황표 자동대조(등식1): 별첨 값과 "한장 보장현황 표"를 따로 읽어 서버가 기계 대조. 일치=✅ / 불일치=⚠ 담보별 표시+메모, 받기 전에 보인다.
6. 회사명 3단 복원: ①목록 회사열 원문 ②별첨 제목줄 ③상품명에서 서버 복원. 앞번호 제거, 끝까지 없으면 "미확인" — 추측 금지.

## Ⅲ. 담보 매핑
7. 표준 담보명 사전 사용 + 별칭 자동변환 2차 방어.
8. 절대 표 밖 금지(예외 없음) — 폼 외 담보는 해당 그룹 블록 안에 행 삽입(a열 병합 자동 확장), 새 그룹은 표 끝에 a열 구분으로 연결.
9. 1~5종 수술비 = 질병/상해 각 1행 가로 "20/30/300/1000/1000", 합계도 종별 합산.
10. n대수술비=대표 1행. 통합암·전이암=대표 1개. 하이클래스=비급여주요치료비. 제외: 5/7대 장기이식·조혈모·각막. 간병인지원=일당 칸 파랑.
11. 심장 = 담보명 직행(허혈성→허혈성 칸, 전파 금지). 특정Ⅰ/Ⅱ/Ⅲ 불확정은 그대로 기재+[확인]+약관검색 링크. 칸 순서: 부정맥·심부전·염증·협심증·빈맥·심근병·판막·급성심근경색·허혈성.

## Ⅳ. 갱신/비갱신 — 서버 직접 계산
12. ①"갱신형" 명시→갱신 ②운전자→비갱신 ③9999→비갱신 ④총회차 240 초과→비갱신 ⑤납입년수≠보장년수→비갱신, 일치→갱신. ("비갱신암" 단어에 안 속음)
13. 제목칸: 갱신=블루/비갱신=레드/완납=짙은녹색, 글자 흰색. 값글자: 갱신=파랑/비갱신=검정, 미가입=빨강. 숫자 콤마. 특약 "(갱신)"은 그 칸만 파랑(칸>행).
14. 9999(생보)=종신 → 일반사망(종신)+상해사망 1:1. 손보 질병사망=80세 칸.

## Ⅴ. PPT·메모·검수
15. PPT는 완성된 엑셀만 읽는다(등식2). 매핑 안 되는 값은 비우고 메모.
16. 메모는 [확인]/[부족]/[미가입] 태그별 구분 + 선·여백 + 카테고리 묶음. 깨알 한 줄 나열 금지. 각 줄 약관검색 원클릭.
17. 검수채팅 + "팩폭" 상시. 다음 단계: 채팅 수정 명령 → 파일 재생성.
18. 추측 금지 헌장: 채우는 게 최악, 비우고 메모가 차선, 확정 기재가 최선. 자동 천장 ~95%, 나머지 5%는 자수하고 사람이 닫는다.

## Ⅵ. 신규 (06.05 추가)
19. 절대 표 밖 금지 — 8조의 헌법화, 예외 없음.
20. PPT 폼 무변형 — 도형·폰트·크기·위치 불가침, 값 텍스트만 삽입.
21. 삭제 금지·추가만 — 전 산출물 공통.
22. 못해요 플랜(정본 확정): 뇌혈관/심장/암/기타 4구획. 못 채운 항목을 🔵블루=확인 후 확정 / ⚫블랙=가입 없음(보완 제안 가능)으로 기재. 이 색은 13조 표 색과 역할 분리(표=갱신색 체계, 플랜=확인색 체계). 구획 초과 시 "외 N건(엑셀 메모시트)" 절단.

## Ⅶ. 폼 구성 확정
- 폼 3개 개념 = 파일 2개: template.xlsx(엑셀 표) + form.pptx 3페이지(한장보장 상급암주치 1·2p + 못해요플랜 3p).
- 분석 1회 = 엑셀 1개 + PPT 1개(합본) 다운로드.
- form.pptx는 PPT 재작성 완료 시 fill_pptx.py와 동시 업로드 — 그 전 단독 업로드 금지(옛 코드가 부숨).
