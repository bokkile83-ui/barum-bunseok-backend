# 바름 보장분석 자동화 — 법칙 정본 (한글)
업데이트: 2026.06.08 / 엔진: engine_v2.py + rules.py / 대상: 두호 14계약

이 문서는 코드(rules.py·engine_v2.py)에 박힌 법칙을 사람이 읽는 한글로 옮긴 것이다.
법칙이 흔들리지 않도록 **코드가 정본**이며, 이 문서는 그 거울이다.

---

## Ⅰ. 추출 엔진 (절대 규칙)
1. **추출 엔진 = PyMuPDF(fitz) 단독.** poppler 금지(OZ뷰어 Adobe-Korea1 CID폰트라 한글 0개).
2. **별첨 직독 = get_text 줄 단위(담보명 → 바로 다음 숫자 = 값).** 값 검증은 좌표(get_text words)로.
3. **'~' 필터 예외:** 추출 시 '~'가 든 줄은 기간(2026~2046)이라 버리되, **'종'이 들어간 줄은 살린다**(질병1~5종·상해1~5종 담보 보호). ← 이게 빠지면 1-5종 통째 누락.
4. **회사·상품명 = 별첨 페이지 헤더 1·2줄.** PAGES dict로 계약열 → 별첨페이지 지정.

## Ⅱ. 등식·폼 (손대지 말 것)
5. **등식: PDF표 = 엑셀 = PPT.** 완성 엑셀이 단일 출처.
6. **표준 폼 무변형.** 새 워크북 발명·행삭제·카테고리축소 금지. **행 추가만** 허용, 추가한 칸은 다시 손대지 않는다.
7. **이번에 영구 추가된 행:** 일반암(갱신형) / 중입자치료 / 116대수술비.
8. **동적 행매핑:** 행은 폼 B열 담보명으로 찾는다(NOW·REMAP). 폼에 행을 더해도 아래 행이 밀려 깨지지 않는다(시프트 버그 영구 차단).

## Ⅲ. 갱신 판정 (법칙12)
9. 순서대로 첫 매칭: ① 담보명에 '갱신형' → 갱신 ② 만기 9999(종신) → 비갱신 ③ 총회차 > 240 → 비갱신 ④ 납입년수 = 보장년수 → 갱신 / 다르면 비갱신.
10. **'운전자 → 비갱신' 규칙은 삭제됐다.** 운전자보험도 납입=보장이면 갱신(예: DB손보 참좋은운전자 2510, 20년납=20년만기 → 갱신).

## Ⅳ. 색 (법칙13)
11. 값: **갱신 = 파랑 / 비갱신 = 검정 / 미가입 = 빨강.**
12. 헤더: 갱신 = 파랑 / 비갱신 = 빨강 / 완납 = 진녹 / 글자 흰색.
13. **계약(열)이 갱신이면 그 열의 모든 담보 값을 파랑으로 통일.** 비갱신 계약은 담보별(갱신형 담보만 파랑).
14. **항암치료 신담보 갱신강제(파랑):** 표적항암·세기조절·중입자·양성자·카티는 별첨 '갱신형' 표기가 잘려도 파랑으로 본다.

## Ⅴ. 사망·암 (법칙14·23·24·25)
15. **종신(9999) = 일반사망 + 상해사망** 양면 1:1 기재.
16. **일반암 = 비갱신/갱신형 행 분리.** 한 칸 슬래시 금지. 일반암(비갱신, 검정) / 일반암(갱신형, 파랑) 별행. 검산은 두 행 합.
17. **유사암:** 별첨 기재 시 직독. 미기재 시 가입연도 ≤2020 → 일반암÷10, ≥2021 → 일반암÷5.
18. **표적항암 = 대표 1건(최댓값).** 여러 표적담보 있어도 한 칸 최댓값.
19. **중입자 = '항암중입자방사선치료비'만.** 카티(CAR-T)는 중입자가 아니다 → 중입자 행에 넣지 않는다.
20. **항암방사선약물 = 항암방사선치료비 + 항암약물치료비(허가 제외).** '주요치료비'는 누수 차단(암주요치료비로 분리).

## Ⅵ. 뇌혈관 (법칙29)
21. **라이나생명(N·O열) 별첨 '뇌혈관진단'은 실제 '뇌출혈'(한장보장표 기준) → 뇌출혈진단비 행으로 교정.**
22. 단, 한 계약에 뇌혈관진단·뇌출혈진단이 둘 다 실제 가입돼 있으면(예: 라이나 C열) 그대로 둔다.

## Ⅶ. 수술 (법칙 신규)
23. **상해수술비 = 기본만.** '상급종합병원·종합병원' 상해수술비는 병원규모 가산이라 기본 상해수술비에 합산하지 않는다(제외).
24. **1-5종 수술비 = 질병/상해 각 1행, 종별 슬래시(1종/2종/3종/4종/5종).** 종번은 담보명의 '_N종' 또는 '(N종'에서 추출.
25. **116대수술비 = 1-5종(5단계)보다 상위 등급.** Ⅰ~Ⅵ 6단계를 별도 행에 단계별 슬래시(예 1000/500/200/100/30/30). 1-5종과 절대 섞지 않는다.

## Ⅷ. 표기·검산 (법칙 공통)
26. **월보험료·합계 = 천단위 쉼표(#,##0).**
27. **추측 금지.** 모르면 빈칸 + [확인]. 별첨 직독값이 진실.
28. **4P 한장보장표 = 검산용(세로 SUM = 4P면 ✅).** 4P 값을 합계칸에 직박는 눈속임 금지. 검산 라벨이 틀려도 고객 표 담보값(별첨 직독)은 별개로 정확.
29. **결과물 주기 전 이 점검표 통과 필수.** 하나라도 어기면 주지 않는다.

## Ⅸ. PPT 보장분석지 (make_ppt.py)
30. **완성 엑셀(engine cell) → 폼(폼_보장분석지_17p.pptx) 빈 1장에 라벨별 자동 기입.** 엑셀=PPT 단일출처.
31. **색: 비갱신=검정 / 갱신=파랑 / 미가입=빨강.** 담보별 비갱신합·갱신합을 분리 표기(예 "2000/25300", 검정/파랑).
32. **1~5종·116대수술 = 슬래시.** 라이나(C·N·O) 뇌혈관진단 = 뇌출혈로 합산(법칙29).
33. **칸 넘침 = 값 폰트 0.8 축소** + 제목 한 줄. 표(폼) 구조는 손대지 않는다.
