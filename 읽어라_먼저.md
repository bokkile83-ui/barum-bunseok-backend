# 1. CI 3-state 검증 결과 (복구본 실행)

| 상황 | status | 결과 |
|---|---|---|
| CI상품 + 중대한OO 추출됨 | `ci` | 선지급률 80% · 사망기준 4,000만 · 잔여 800만 |
| CI상품 + 중대한OO 미추출 | `check` | P3 골드 「CI 판정 보류 [확인]」 · Plan B 낙하 안 함 |
| 비CI 상품 | `none` | 「CI보험 아님」 |

**CI 로직에 버그 없다.** 파일이 지워져서 안 돌았을 뿐이다.

## 김진구님 케이스
교보생명 「무배당교보큰사랑 CI 보험」.
별첨 담보명이 전부 **「주계약」** 이라 `중대한OO` 를 뽑을 수 없다.
- 주계약 4,000 = 일반사망(종신)
- 주계약 3,200 × 4건 = CI 선지급 80% (4,000 × 0.8)
- CI 추가보장특약 200 / 1,000

→ 정상 판정 = **`check` (골드 [확인])**.
수기로 `중대한 암 3,200 / 중대한 뇌졸증 3,200 / 중대한 급성심근 4,200 / 중대한CI적용 800`
입력하면 `ci` · 선지급률 80% 로 확정된다.

# 2. 신규 발견 — 엑셀 수식 캐시

`coverage_benchmark.load_excel` 은 끝열을 `data_only=True` 로 읽는다.
**수식 캐시가 없으면 전 담보 0** 이 된다.

openpyxl로 엑셀을 재저장하면 캐시가 날아간다.
→ 동봉한 `보장진단_*_보정.xlsx` 는 LibreOffice 재계산으로 **캐시를 복원해 두었다.**
   앱에 그대로 넣어도 된다.
