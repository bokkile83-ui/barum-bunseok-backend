# report_pptx.py v35 — 뼈대 유지, 값 칸만 입력·수정

## 무엇이 바뀌나
```
① report_weasy 로 보장설명서 PDF 생성        ← 07.07 디자인 원본
② 각 페이지를 200dpi 이미지로 → 슬라이드 배경  ← 뼈대·라벨·표·게이지 그대로
③ rep 에서 '결과값' 문자열 45개를 뽑는다
④ pdftotext -bbox-layout 으로 그 값의 좌표만 찾는다
⑤ 그 자리만 배경색으로 지우고 텍스트 상자를 얹는다
```

**라벨·제목·표 선·게이지 링·심혈관 분류표는 손대지 않는다. 값만 클릭·수정된다.**

# 검증 (김진구님 실제 PDF·엑셀)

## 원본 PDF와 픽셀 일치율
```
 1p 99.5   2p 97.4   3p 97.7   4p 97.0   5p 95.6
 6p 96.2   7p 94.1   8p 97.7   9p 95.1  10p 95.3
```

## 실제 값 수정 → 저장 → 재렌더 → 글자 재추출
| 고친 값 | 결과 |
|---|---|
| 80% → 50% | 렌더 PDF에 `50%` 8회 ✓ |
| 3,200만 → 5,000만 | `5,000만` 16회 ✓ |
| 6 건 → 9 건 | ✓ |
| 324,470 원 → 999,999 원 | ✓ |
| 2억9,200만 → 7억7,777만 | ✓ |

옛 값 7개 전부 렌더에서 사라짐 ✓

## 뼈대 보존
렌더 PDF에서 `CI 선지급 분석`·`한화손해보험`·`급성 심장막염` 이 글자로 안 잡힌다.
→ 배경 이미지라 그렇다. **정상이고, 그게 증거다.**

## 구조
```
그림이 맨 아래 → 글자가 위     클릭하면 잡힘
잠금(spLocks) 0개               선택 막힘 없음
겹침 0쌍                        엉뚱한 게 안 잡힘
폰트 맑은 고딕 (latin + ea)     안 깨짐
값 상자 오른쪽 여유 중앙값 200pt
```

## 극단 입력
`80%` 칸(3글자)에 `12억3,456만7,890`(14글자) 입력 → 정상 렌더.

# 아직 확인 못 한 것 (자진보고)
- **파워포인트 UI에서 마우스 클릭·키보드 입력.** 나는 파워포인트를 못 연다.
- 눈으로 본 적 없다. 전부 숫자로만 쟀다.
- 값을 고쳐도 **게이지 링·표 선은 배경**이라 안 바뀐다. 값 바뀌면 재생성해야 한다.
- 자동 줄바꿈 없음. `10만`·`1억`·`3,000만` 6칸은 오른쪽 여유 50pt 미만.

**먼저 `보장진단서_김진구_v35.pptx` 를 열어서 `80%` 를 클릭해봐라.**
