# 질병수술비 합산 버그 패치 (50→750 재발 방지)

## 원인
배포 v27b 코드가 `has('질병수술비')` 부분일치로 매칭 →
① 변형 질병수술비(Ⅱ·간편·갱신 등)가 같이 합산
② 종(1~5종) 미감지 시 슬래시 금액이 통째 합산(50+100+150+200+250=750)

## 지침(v29 §8.5)
질병수술비 합산군 = '질병수술비'·'질병입원수술비'·'질병수술비(**제외)' 3종만.
그 외 **대수술비·특정/부위/관절/종합병원 등 변형·질병수술일당 = 개별, 합산 금지.

## main.py 매처 교체 (질병 분기)
WL_JBSS = ('질병수술비','질병입원수술비')
if has('질병'):
    norm=_norm(r)
    is_excl = norm.startswith(_norm('질병수술비(')) and '제외' in norm
    is_wl   = norm in [_norm(x) for x in WL_JBSS] or is_excl
    if is_wl and '/' not in str(amt) and no('특정','부위','관절','척추','외모','흉터','복원','신경','인대','연골','상급','종합','대수술','일당'):
        return '질병수술비',0
    return '[확인]질병수술비변형',0   # 변형·종·대수술비 → 개별/확인

## 핵심
1) 정확일치 3종만  2) 슬래시(종) 값이면 진입 차단  3) 변형은 [확인]로 분리(합산 제외)
